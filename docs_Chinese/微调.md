# FR3 LoRA 微调

本文档对应 `src/openpi/training/config.py` 里的 `pi05_fr3_lora` 配置：以本地 `models/pi05_droid` 为 base checkpoint，在 FR3 采集的 LeRobot v2.1 数据集上做 LoRA 微调。

## 1. 前置目录

在仓库根目录准备：

```text
models/pi05_droid/
  params/
  assets/

datasets/fr3_pick_up_cup/
  data/
  videos/
  meta/
```

`models/pi05_droid/assets` 里的 DROID norm stats 会被复用。当前 FR3 数据配置会把 FR3 的字段 repack 成 DROID/pi0.5 期望的字段。

数据集目录名要和训练配置里的 `repo_id` 对上。当前配置是：

```python
repo_id="fr3_pick_up_cup"
```

所以本地路径应是：

```text
datasets/fr3_pick_up_cup
```

如果你换了任务名，比如 `datasets/fr3_pick_block`，需要同步改 `src/openpi/training/config.py` 里的 `repo_id`，或者把数据目录改名。

## 2. 启动训练

```bash
cd /path/to/openpi
conda activate openpi-dev

export HF_LEROBOT_HOME="$(pwd)/datasets"
export HF_HUB_OFFLINE=1
export XLA_PYTHON_CLIENT_MEM_FRACTION=0.9

python scripts/train.py pi05_fr3_lora --exp-name <run_name> --overwrite
```

训练输出默认在：

```text
checkpoints/pi05_fr3_lora/<run_name>/<step>/
```

例如：

```text
checkpoints/pi05_fr3_lora/test_train_2/199999/
```

## 3. 当前配置要点

`pi05_fr3_lora` 当前使用：

- `pi05=True`
- `action_dim=32`
- `action_horizon=16`
- `paligemma_variant="gemma_2b_lora"`
- `action_expert_variant="gemma_300m_lora"`
- `weight_loader="models/pi05_droid/params"`
- `assets_dir="models/pi05_droid/assets"`
- `ema_decay=None`
- `wandb_enabled=False`

LoRA 训练必须保留对应的 `freeze_filter`，否则会变成全参训练或参数集合不匹配。当前配置已经用 `get_freeze_filter()` 处理好了。

FR3 的第 7 关节在数据 transform 和推理端都做了同一个 offset：

```text
joint_position[6] -= pi / 4
```

所以训练和真机推理要使用同一套代码，不要只改一边。

## 4. 常用调参位置

在 `src/openpi/training/config.py` 的 `pi05_fr3_lora` 配置里改：

- `num_train_steps`：总步数。
- `batch_size`：显存不够先降它。
- `save_interval`：checkpoint 保存间隔。
- `keep_period`：保留周期 checkpoint。
- `wandb_enabled`：需要线上记录时再打开。

小数据集先建议多看验证/真机效果，不要只盯 loss。过拟合时可以减少步数、增加数据多样性，或把同一任务的不同初始位姿补齐。

## 5. 断点续训

继续同一个实验：

```bash
python scripts/train.py pi05_fr3_lora --exp-name <run_name> --resume
```

不要同时加 `--resume` 和 `--overwrite`。

## 6. 启动微调后的策略服务

训练产出的完整 checkpoint 可以直接作为 `--policy.dir`：

```bash
export XLA_PYTHON_CLIENT_PREALLOCATE=false
export XLA_PYTHON_CLIENT_MEM_FRACTION=.70

python scripts/serve_policy.py \
  --port 8010 \
  policy:checkpoint \
  --policy.config=pi05_fr3_lora \
  --policy.dir=checkpoints/pi05_fr3_lora/<run_name>/<step>
```

然后按 `docs/真机部署.md` 启动 `inference_client.py`。

如果你只想保存/分发 LoRA adapter，或者要把 adapter 合并回 base checkpoint，见：

```text
docs/LoRA合并与拆分.md
```

## 7. 训练前检查

```bash
python -c "import openpi, cv2, pyarrow; print('env ok')"
test -d models/pi05_droid/params
test -d models/pi05_droid/assets
test -d datasets/fr3_pick_up_cup/meta
```

如果 LeRobot 找不到数据，大概率是 `HF_LEROBOT_HOME` 没指到仓库下的 `datasets`，或者 `repo_id` 和目录名不一致。
