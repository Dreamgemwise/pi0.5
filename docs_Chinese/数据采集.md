# 数据采集

这里记录 FR3 上给 `pi05_fr3_lora` 微调用的数据采集流程。数据会落成 LeRobot v2.1 格式，字段名对齐 DROID，这样可以直接走 OpenPI 的 DROID/pi0.5 输入输出变换。

## 1. 数据流

采集 RT 端负责通过 Polymetis 只读机器人状态，并发布 100Hz robot state：

```text
Franka/FrankaTeleop/Polymetis
  50051: robot state gRPC
  50053: gripper state gRPC
        |
        v
x86 RT 采集机 readonly_state_publisher.py
  5555: RobotState PUB  --->  采集 PC
```

采集 PC 负责：

- 订阅最新 robot state。
- 读取两路 ZED UVC 单目 RGB。
- 按 15Hz 写 LeRobot v2.1 数据集。
- 用终端热键控制 episode 开始、保存、丢弃。

纯采集时不需要策略服务，也不需要 OpenPI 推理。采集 RT 端也不要启动 `robot_server.py`，因为采集只需要读状态，不需要打开 action 接收端口或给机器人发命令。

注意：这一块只能用 `examples/fr3_realtime/readonly_state_publisher.py`。它依赖 Polymetis gRPC 服务，多端口由 FrankaTeleop/Polymetis 环境挂载；Polymetis 只能运行在 x86 上，所以数据采集 RT 机必须是 x86 主机，不能用 ARM/C++ libfranka 那条链路。

## 2. 启动 x86 采集 RT 机

先在 x86 RT 主机上按 FrankaTeleop 的方式启动 Polymetis/FrankaTeleop，让 robot state 和 gripper state 服务可用。环境使用 FrankaTeleop 创建的 `polymetis` 环境：

```bash
conda activate polymetis
```

然后在同一台 x86 RT 主机上，从 `examples/fr3_realtime` 启动只读状态发布程序：

```bash
cd examples/fr3_realtime

python readonly_state_publisher.py \
  --polymetis-host 127.0.0.1 \
  --polymetis-port 50051 \
  --gripper-port 50053 \
  --bind 0.0.0.0
```

如果没有夹爪状态服务，或者暂时不读夹爪状态，可以关闭 gripper 读取：

```bash
python readonly_state_publisher.py \
  --polymetis-host 127.0.0.1 \
  --polymetis-port 50051 \
  --gripper-port 0 \
  --bind 0.0.0.0
```

`readonly_state_publisher.py` 只读取 Polymetis gRPC state，并把 `RobotState` 发布到 `tcp://<bind>:5555`。它不会导入 `polymetis.RobotInterface`，不会进入 impedance control，也不会打开 action socket。

## 3. 预检查相机和网络

在采集 PC 上：

```bash
conda activate openpi-dev

ping <RT-PC-IP>
v4l2-ctl --list-devices
ls /dev/video*
```

这里的 `<RT-PC-IP>` 是运行 `readonly_state_publisher.py` 的 x86 RT 主机 IP。

预览 front 或 wrist 相机：

```bash
python scripts/zed_preview.py --camera /dev/video0 --camera-eye LEFT --camera-fourcc YUYV
python scripts/zed_preview.py --camera /dev/video2 --camera-eye LEFT --camera-fourcc YUYV
```

如果腕部相机画面倒置，采集命令里加 `--flip-wrist`；如果外部相机倒置，加 `--flip-front`。

## 4. 纯数据采集

在采集 PC 的仓库根目录运行：

```bash
python examples/fr3_realtime/data_collector.py \
  --robot-ip <RT-PC-IP> \
  --front-camera /dev/video0 \
  --wrist-camera /dev/video2 \
  --prompt "pick up the cup" \
  --record-dir ./datasets/fr3_pick_up_cup \
  --flip-wrist \
  --display
```

热键：

```text
Enter = 开始一集
s     = 停止并保存当前集
r     = 丢弃当前集
q     = 退出
```

采集时按住 FR3 末端 guiding 按钮拖动演示，或让其他专家控制器驱动机器人。脚本会用相邻两帧关节角差分得到关节速度，作为训练 action。

常用相机参数：

```bash
# 默认 HD720：2560x720@30fps，单目约 1280x720
python examples/fr3_realtime/data_collector.py ... --camera-mode HD720

# 更高清但只有 15fps，单目约 2208x1242
python examples/fr3_realtime/data_collector.py ... --camera-mode HD2K

# 取右目
python examples/fr3_realtime/data_collector.py ... --camera-eye RIGHT
```

## 5. 采集链路限制

本文档这条 FrankaTeleop/Polymetis 采集链路只使用 `readonly_state_publisher.py`。不要在采集 RT 机上用 `robot_server.py` 代替它，也不要用 ARM/C++ libfranka 版作为采集 state bridge。

`inference_client.py --record-dir` 属于真机执行/策略闭环时的记录方式，需要执行端 `robot_server` 接收 action chunk；它不是本文的 Polymetis 纯采集流程。

## 6. 数据集目录结构

保存后目录大致是：

```text
datasets/fr3_pick_up_cup/
  data/chunk-000/episode_000000.parquet
  videos/chunk-000/observation.images.exterior_image_1_left/episode_000000.mp4
  videos/chunk-000/observation.images.wrist_image_left/episode_000000.mp4
  meta/info.json
  meta/tasks.jsonl
  meta/episodes.jsonl
  meta/episodes_stats.jsonl
```

关键字段：

- `observation.state`：8 维，`joint_position(7) + gripper_position(1)`。
- `action`：8 维，`joint_velocity(7) + gripper_position(1)`。
- `observation.images.exterior_image_1_left`：外部 ZED 单目 RGB。
- `observation.images.wrist_image_left`：腕部 ZED 单目 RGB。

采集频率默认 `15Hz`，对齐 DROID/pi0.5。

如果 `episodes_stats.jsonl` 丢了或需要重建：

```bash
python scripts/gen_episodes_stats.py --root datasets/fr3_pick_up_cup --overwrite
```

## 7. 质量检查

每个 episode 建议保持单个清晰任务，不要把失败、空闲等待、重新摆物体混进同一集。失败演示可以单独保存，但训练前要决定是否纳入。

检查重点：

- prompt 和动作语义一致。
- front/wrist 两路画面没有左右目取错、没有倒置。
- 机器人拖动过程尽量连续，少停顿。
- 数据目录名和 `src/openpi/training/config.py` 里的 `repo_id` 对上，例如 `HF_LEROBOT_HOME="$(pwd)/datasets"` 时，`repo_id="fr3_pick_up_cup"` 会读 `datasets/fr3_pick_up_cup`。
